## 5. 入力データ・マスター情報

| データ名 | 現在の形式 | 必須列 | 任意列 | 更新頻度 |
| --- | --- | --- | --- | --- |
| 利用者マスター | Excel / CSV / 紙 | 氏名、住所 | 車いす、曜日、備考 |  |

| 出力帳票 | Excel / 紙 | 車両、順番、利用者 | 注意事項 |  |

CSV取り込み時の確認:

- 文字コードはUTF-8またはShift_JISのどちらか
- 氏名の表記揺れがあるか
- 住所が番地まで入っているか
- 同姓同名の区別方法があるか
- 古い利用者が混ざっていないか

## 6. 画面設計

画面ごとに、誰が何をする画面なのかを決めます。

| 画面名 | 使う人 | 主な目的 | 必須操作 | 表示したい情報 |
| --- | --- | --- | --- | --- |
| ログイン | 全員 | 本人確認 | PIN入力 | 利用者名 |
| 顧客登録 | 管理者 | 利用者マスター登録 | 登録、編集、CSV取込 | 氏名、住所、曜日、車いす |
| 当日一覧 | 管理者 / 現場 | 欠席・早帰り・追加 | 欠席、時間変更、当日追加 | 今日来る人 |
| 地図・ルート | 管理者 / ドライバー | 走行順確認 | ルート作成 | 地図、到着目安 |
| 配車表 | 管理者 / ドライバー | 車両別共有 | CSV、コピー、印刷 | 車両別の順番 |

UI/UX確認:

- 操作する人の年齢層
- 画面上に説明文を多く置くべきか、ボタン名を分かりやすくすべきか
- スマホ片手操作が必要か
- 車内で見るため文字を大きくする必要があるか
- 誤操作防止の確認画面が必要か

優先度の基準:

## 9. セキュリティ・個人情報

| 確認項目 | 方針 |
| --- | --- |
| ログイン方式 | 管理者・担当者ごとのアカウント |
| パスワード/PIN | 初期値を本番前に変更 |
| 権限 | 管理者、編集者、ドライバー、閲覧のみを分ける |
| 操作ログ | 登録、変更、削除、閲覧を記録 |
| 保存期間 | 業務上必要な期間を決める |
| バックアップ | 毎日または週次で保存 |
| 削除ルール | 退所者、退職者、誤登録の扱いを決める |

個人情報を扱う場合の確認:

- 本名と住所を保存する必要があるか
- 電話番号、病歴、介護度などを保存するか
- ドライバーに見せる情報を最小限にできるか
- CSV出力ファイルの保管先を決めているか

## 10. 出力・帳票

| 出力名 | 形式 | 使う人 | 用途 | 必須項目 |
| --- | --- | --- | --- | --- |
| 配車表 | CSV / Excel貼り付け / 印刷 | ドライバー | 当日運行 | 車両、順番、氏名、住所、注意 |
| 管理表 | CSV | 管理者 | 確認・保管 | 日付、利用者、変更内容 |
| 操作ログ | CSV | 管理者 | 監査 | 日時、操作、担当者 |

確認すること:

- 1車両ごとに分ける必要があるか
- 1枚の紙にまとめたいか
- 備考欄をどこまで入れるか
- 紙を使わずスマホで確認する運用にするか

## 11. 受け入れ条件

納品前に以下を確認します。

- 管理者がログインできる
- 登録済みデータを編集できる
- CSV取り込みができる
- 主要画面がPCとスマホで崩れない
- 権限のない人が重要情報を編集できない
- 出力帳票を現場で使える
- 当日変更が保存される
- 操作ログが残る
- 本番URLで確認できる

## 12. 変更要望の扱い

開発途中で新しい要望が出た場合は、次のどれに該当するか確認します。

| 区分 | 内容 | 対応 |
| --- | --- | --- |
| 仕様の明確化 | 最初の目的に含まれているが詳細が未確定 | 初回範囲で調整 |
| 仕様変更 | 最初に決めた動きと異なる | 工数・納期を再確認 |
| 機能追加 | 新しい画面や出力が必要 | 追加開発として扱う |
| 本番障害 | 保存できない、ログインできない等 | 優先対応 |

## 13. 介護送迎ルートアプリ専用ヒアリング

### 13-1. 車両情報

| 車両 | 定員 | 車いす対応 | 車いす台数 | 備考 |
| --- | --- | --- | --- | --- |
| 1号車 |  | あり / なし |  |  |
| 2号車 |  | あり / なし |  |  |
| 3号車 |  | あり / なし |  |  |
| 4号車 |  | あり / なし |  |  |

確認すること:

- 車いす仕様車に車いす利用者が乗らない場合、普通席運用してよいか
- 車いす利用者はどの車両に乗れるか
- 車いす利用者がいる場合、乗車順・固定位置・降車順に制約があるか
- 定員には職員や付き添いを含めるか

### 13-2. 利用者マスター

CSVまたはExcelで受け取る推奨列:

| 列名 | 必須 | 説明 |
| --- | --- | --- |
| 利用者名 | 必須 | 氏名または呼称 |
| 住所 | 必須 | 送迎先住所 |
| 人数 | 任意 | 通常は1 |
| 車いす | 任意 | あり / なし |
| 何時以降 | 任意 | 迎え・送りの開始希望 |
| 何時まで | 任意 | 迎え・送りの終了希望 |
| 利用曜日 | 任意 | 月,水,金 など |
| 帰り区分 | 任意 | 通常 / 13:30 |
| 備考 | 任意 | 玄関、声かけ、家族連絡など |

### 13-3. 当日運用

| 当日変更 | 必要な操作 |
| --- | --- |
| 明日来ない | 対象者を欠席にする |
| 通常16:30帰りから13:30帰りへ変更 | 帰り区分を変更する |
| 当日だけ来る | 当日一覧で利用者を追加する |
| 予定時刻に乗らない | 配車表備考へ理由や対応を書く |
| 迎えを何時以降にしてほしい | 当日一覧で迎え希望開始を入れる |
| 送りを何時以降にしてほしい | 当日一覧で送り希望開始を入れる |

### 13-4. ルート作成の完成条件

- CSVで住所一覧を取り込める
- 登録済み利用者から当日搭乗者を作れる
- 住所をもとに地図上へ表示できる
- 車両定員を超えないように確認できる
- 車いす利用者は車いす対応車両へ割り当てられる
- 車いす仕様車を普通席運用する場合は配車表に表示される
- 迎え・送りどちらも「何時以降」「何時まで」を利用できる
- 各車別にドライバーへ渡せる配車表を出力できる

6. 出力帳票の形
