# FM大師 業務管理

FM大師向けのシフト作成・共有・交代承認、放送資料フォルダー、CD台帳、選曲カレンダーをまとめた業務管理ウェブアプリです。Node.js標準機能だけで動くため、追加パッケージなしで試せます。

## 起動方法

```bash
npm start
```

または:

```bash
node server.js
```

ブラウザで `http://localhost:8000` を開きます。

実運用でスタッフ全員が同じデータを更新する場合は、`npm start` で動く Node.js 版を Render、VPS、社内サーバーなどに配置してください。

## 本番サーバー運用

患者様情報を扱う場合、GitHub Pagesの静的URLではなく、Node.jsサーバー版をHTTPS付きで公開してください。

本番起動に必須の環境変数:

```bash
NODE_ENV=production
CARE_ROUTE_DATA_KEY="32文字以上のランダムな秘密キー"
LOGIN_PINS='{"shimizu":"123456","suyama":"234567","customer_demo":"345678"}'
```

`CARE_ROUTE_DATA_KEY` は `openssl rand -base64 48` などで生成し、GitHubやコードには保存しません。本番ではこのキーが未設定、または32文字未満の場合はサーバーが起動しません。

PINは必要なユーザーだけ設定できます。個別指定する場合は `PIN_SHIMIZU`、`PIN_SUYAMA`、`PIN_CUSTOMER_DEMO` のように設定できます。

本番で追加される安全設定:

- コード内の開発用PINを無効化
- 6文字未満の本番PINを拒否
- 期限付きセッション: 初期値12時間
- ログイン失敗の回数制限: 初期値10分8回
- ログアウト時のサーバーセッション破棄
- JSON/API/静的配信への基本セキュリティヘッダー
- 介護送迎データのAES-256-GCM暗号化保存

Renderで運用する場合は `render.yaml` を使い、`CARE_ROUTE_DATA_KEY` と `LOGIN_PINS` を管理画面の環境変数に設定してください。DockerやVPSで動かす場合は `Dockerfile` と `.env.production.example` を参考にしてください。

## スタッフ画面と管理者画面

スタッフは、月間シフト、日別一覧、自分のシフト、希望提出、掲示板、通知履歴だけを使うシンプルな画面です。

管理者である清水 暁さん、須山 成美さんだけが、仮シフト共有、確定、シフト編集、承認、CSV取込、番組枠マスタ、スタッフ情報、書類送付、加盟店管理を操作できます。

## PWAで社内運用する流れ

PWAは、App StoreやGoogle Playに出さずに、スタッフのスマホのホーム画面へ追加して使う運用です。

### まず社内Wi-Fiで試す

このMacでサーバーを起動した状態で、同じWi-Fiにつながっている別PCやスマホから次の形式で開きます。

```text
http://MacのIPアドレス:8000/
```

例:

```text
http://192.168.10.103:8000/
```

### ホーム画面へ追加

- iPhone: Safariで開き、共有ボタンから `ホーム画面に追加`
- Android: Chromeで開き、画面上の `ホームに追加` またはブラウザメニューから追加

アプリ内にも `ホームに追加` ボタンを表示します。iPhoneではボタンを押すと、Safariの共有ボタンから追加する案内が出ます。

### 本番運用で必要なこと

社内全員が安定して使うには、HTTPS付きのURLで公開する必要があります。

- クラウドサーバーへ配置
- 独自ドメインまたは固定URL
- HTTPS化
- `data/store.json` のバックアップ
- PINではなく正式なログイン方式への移行

## AIで作成したシフトCSVの取り込み

管理者でログインし、「AI/CSV連携」から使います。

1. `AI用プロンプトをコピー`
2. AIへ貼り付けてCSVを作成
3. 必要なら `CSVテンプレート出力` または `標準形式CSV出力` の形式に合わせて調整
4. `CSVファイル` を選択、または `CSV貼り付け` にAIの出力をそのまま貼り付け
5. `CSV内容を確認`
6. 変更点と警告を確認し、問題なければ `プレビューを反映`

AI用の基本CSV列は `日付,担当枠,主担当,補助担当` です。日付は `2026-06-03` のような形式、担当枠は `09:00枠` から `17:00枠`、または `9時` から `17時` の表記を使えます。

Excel原本に近い横持ち形式も取り込めます。列は `日付,曜日,名前,09:00~,10:00~,11:00~,12:00~,13:00~,14:00~,15:00~,16:00~,17:00~,オリジナル番組` です。時間列にはスタッフ名またはスタッフ記号を入れます。`名前` 列にスタッフ名を入れて、時間列を `○` にする形式にも対応しています。

現時点では、ChatGPTやGeminiをアプリ内から直接呼び出すのではなく、APIキーをアプリに保存しない安全な形として「AIにCSVを作らせて、ファイルまたは貼り付けでアプリへ読み込む」方式です。
将来的にOpenAI APIやGemini APIを組み込む場合は、サーバー側でAPIキーを管理し、利用料金、権限、個人情報の扱いを決めてから追加してください。

## 音楽・選曲管理

管理者でログインし、上部タブの `音楽・選曲` から使います。

1. `CD台帳CSVテンプレート` を出力
2. CD写真や動画をChatGPT/Geminiへ渡し、`AI整理プロンプトをコピー` の内容でCSV化
3. `CSV貼り付け` に貼り付け、`CSV内容を確認`
4. 問題なければ `CD台帳に反映`
5. 日付、テーマ、曲数を指定して `選曲を自動作成`

現在の選曲ロジックはAPIキー不要の簡易版です。季節やテーマ、楽曲タグ、直近の利用状況を見て候補を並べます。実運用でOpenAI APIやGemini APIに接続する場合は、APIキーをサーバー側だけに置き、利用料金と個人情報の扱いを決めてから追加します。

## 放送資料フォルダー

上部タブの `放送資料` から、動画台本、実際に話した内容、放送メモを保存できます。

- 放送日、番組名、タイトル、タグ、資料メモ、台本・要約、文字起こしを保存
- 検索で過去資料を確認
- 内容重複、タイトル重複、放送後の文字起こし未入力を自動チェック
- AIチェック用プロンプトをコピーして、ChatGPT/Geminiで精査
- 写真/動画整理プロンプトで、資料をCSV化する準備

## 書類送付管理

管理者でログインし、上部タブの `書類送付` から使います。

- 見積書、納品書、請求書の送付先、件名、金額、発行日、期限日、ファイル名・URLを保存
- 下書き、送付待ち、送付済み、入金済み、取り下げで状態管理
- 一覧の `送付文コピー` でメール本文を作成
- `送付済み`、`入金済み` ボタンでステータス更新
- CSV出力で取引先別・月別の確認資料に利用

現時点では、実メール送信は行わず、メール本文をコピーしてGmail等へ貼り付ける安全な運用です。将来的にGmailやOutlook APIと接続すれば、アプリ内から直接送信する形へ拡張できます。

## 介護送迎ルートの本番セキュリティ

`/care-route/` は、Node.jsサーバーで公開した場合にログイン必須の本番モードになります。GitHub Pagesなど静的公開では本番確認ログインになり、画面構成、入力項目、操作フローを納品直前の状態で確認できます。

- 1号車、2号車は車いす対応車として扱います。
- 1号車、2号車は車いす2台、座席利用5名までで判定します。
- 利用者名と住所は1人ずつ入力できます。
- 顧客登録済みの利用者は、送迎登録フォームのプルダウンから選択すると名前、住所、人数、車いす、時間制約を自動入力できます。
- 名前入力時も、登録済み顧客と一致または一意に近い候補がある場合は住所などを自動入力します。
- 車いす利用者がいる場合、送迎車両は1号車または2号車に制限し、3号車と4号車は選択できない表示になります。
- Excelの一覧からコピーして貼り付ける一括登録にも対応しています。
- 顧客登録はログイン必須です。
- 管理者と配車担当は顧客登録を編集できます。
- ドライバー権限は顧客一覧を取得できず、当日のルート作成だけに使う想定です。
- 顧客データ、削除済みデータ、操作ログは `data/care-route-secure.json` にAES-256-GCMで暗号化保存されます。
- 書き込み時に `data/backups/` へ日次の暗号化バックアップを作成します。
- サーバー運用時は患者名・住所をブラウザの `localStorage` に残さないよう、ルート作成中の一時データは画面メモリだけで扱います。

公開URLでの本番確認ログインは以下です。本番の共通保存、暗号化保存、環境変数管理はNode.jsサーバー版で担保します。

お客様へは次のように説明できます。

> 画面は本番運用を想定した入力項目と操作フローです。本番サーバー版ではログイン、権限管理、暗号化保存、バックアップを有効にして運用します。その状態であれば、新しく利用者を登録しても、保存データは暗号化され、ドライバー権限では顧客一覧を見られない設計です。

| 権限 | 利用者 | PIN |
| --- | --- | --- |
| 管理者 | 管理者 | `0000` |
| 配車担当 | 配車担当 | `1111` |
| ドライバー | ドライバー | `2222` |

本番では必ず以下の環境変数を設定してください。`NODE_ENV=production` で未設定の場合、サーバーは起動しません。

```bash

CARE_ROUTE_DELETED_CUSTOMER_DAYS=30
CARE_ROUTE_ROUTE_DRAFT_DAYS=90

```

## 実運用前の注意

- 本番PINと暗号キーは必ず環境変数で管理してください。
- `data/store.json` は定期的にバックアップしてください。
- `CARE_ROUTE_DATA_KEY` は本番サーバーの環境変数で管理し、GitHubなどに保存しないでください。
- 社外から使う場合はHTTPS、サーバーのアクセス制御、クラウドDBの導入を検討してください。
