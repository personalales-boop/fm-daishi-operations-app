# FM大師 業務管理

FM大師向けのシフト作成・共有・交代承認、放送資料フォルダー、CD台帳、選曲カレンダーをまとめた業務管理ウェブアプリです。Node.js標準機能だけで動くため、追加パッケージなしで試せます。

## 起動方法

```bash
npm start
```

または:

```bash
node server.js
```

ブラウザで `http://localhost:8000` を開きます。

## スタッフ画面と管理者画面

スタッフは、月間シフト、日別一覧、自分のシフト、希望提出、掲示板、通知履歴だけを使うシンプルな画面です。

管理者である清水 暁さん、須山 成美さんだけが、仮シフト共有、確定、シフト編集、承認、CSV取込、番組枠マスタ、スタッフ情報を操作できます。

カレンダー上のスタッフ名をダブルクリックすると、右側の個人別シフト欄が開き、「カレンダーで選択中」と表示され、そのスタッフの今月の担当が一覧で表示されます。

## PWAで社内運用する流れ

PWAは、App StoreやGoogle Playに出さずに、スタッフのスマホのホーム画面へ追加して使う運用です。

### まず社内Wi-Fiで試す

このMacでサーバーを起動した状態で、同じWi-Fiにつながっている別PCやスマホから次の形式で開きます。

```text
http://MacのIPアドレス:8000/
```

例:

```text
http://192.168.10.103:8000/
```

### ホーム画面へ追加

- iPhone: Safariで開き、共有ボタンから `ホーム画面に追加`
- Android: Chromeで開き、画面上の `ホームに追加` またはブラウザメニューから追加

アプリ内にも `ホームに追加` ボタンを表示します。iPhoneではボタンを押すと、Safariの共有ボタンから追加する案内が出ます。

### 本番運用で必要なこと

社内全員が安定して使うには、HTTPS付きのURLで公開する必要があります。

- クラウドサーバーへ配置
- 独自ドメインまたは固定URL
- HTTPS化
- `data/store.json` のバックアップ
- PINではなく正式なログイン方式への移行

## AIで作成したシフトCSVの取り込み

管理者でログインし、「AI/CSV連携」から使います。

1. `AI用プロンプトをコピー`
2. AIへ貼り付けてCSVを作成
3. 必要なら `CSVテンプレート出力` の形式に合わせて調整
4. `CSVファイル` を選択、または `CSV貼り付け` にAIの出力をそのまま貼り付け
5. `CSV内容を確認`
6. 変更点と警告を確認し、問題なければ `プレビューを反映`

CSV列は `日付,担当枠,主担当,補助担当` です。日付は `2026-06-03` のような形式、担当枠は `朝の情報枠`、`昼ニュース`、`夕方番組` のいずれかにしてください。

現時点では、ChatGPTやGeminiをアプリ内から直接呼び出すのではなく、APIキーをアプリに保存しない安全な形として「AIにCSVを作らせて、ファイルまたは貼り付けでアプリへ読み込む」方式です。
将来的にOpenAI APIやGemini APIを組み込む場合は、サーバー側でAPIキーを管理し、利用料金、権限、個人情報の扱いを決めてから追加してください。

## 音楽・選曲管理

管理者でログインし、上部タブの `音楽・選曲` から使います。

1. `CD台帳CSVテンプレート` を出力
2. CD写真や動画をChatGPT/Geminiへ渡し、`AI整理プロンプトをコピー` の内容でCSV化
3. `CSV貼り付け` に貼り付け、`CSV内容を確認`
4. 問題なければ `CD台帳に反映`
5. 日付、テーマ、曲数を指定して `選曲を自動作成`

現在の選曲ロジックはAPIキー不要の簡易版です。季節やテーマ、楽曲タグ、直近の利用状況を見て候補を並べます。実運用でOpenAI APIやGemini APIに接続する場合は、APIキーをサーバー側だけに置き、利用料金と個人情報の扱いを決めてから追加します。

## 放送資料フォルダー

上部タブの `放送資料` から、動画台本、実際に話した内容、放送メモを保存できます。

- 放送日、番組名、タイトル、タグ、資料メモ、台本・要約、文字起こしを保存
- 検索で過去資料を確認
- 内容重複、タイトル重複、放送後の文字起こし未入力を自動チェック
- AIチェック用プロンプトをコピーして、ChatGPT/Geminiで精査
- 写真/動画整理プロンプトで、資料をCSV化する準備

## 書類送付管理

管理者でログインし、上部タブの `書類送付` から使います。

- 見積書、納品書、請求書の送付先、件名、金額、発行日、期限日、ファイル名・URLを保存
- 下書き、送付待ち、送付済み、入金済み、取り下げで状態管理
- 一覧の `送付文コピー` でメール本文を作成
- `送付済み`、`入金済み` ボタンでステータス更新
- CSV出力で取引先別・月別の確認資料に利用

現時点では、実メール送信は行わず、メール本文をコピーしてGmail等へ貼り付ける安全な運用です。将来的にGmailやOutlook APIと接続すれば、アプリ内から直接送信する形へ拡張できます。

## 実運用前の注意

- `server.js` の初期PINは本番前に必ず変更してください。
- `data/store.json` は定期的にバックアップしてください。
- 社外から使う場合はHTTPS、正式なログイン、クラウドDBの導入を検討してください。
